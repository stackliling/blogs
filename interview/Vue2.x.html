# 生命周期

  1. 名称
  • 创建前后 beforeCreate created
  • 挂载前后 beforeMount mounted
  • 更新前后 beforeUpdate updated
  • 销毁前后 beforeDestroy destroyed
  • 激活/失活 activated deactivated

总结：
created之后可以访问数据，如data，computed，methods中的属性或方法
mounted之后可以访问视图，如DOM节点，ref组件实例
beforeDestroy和destroyed时，组件仍可以访问 数据，DOM，方法

  1. 父子组件生命周期顺序
    父子组件创建时：
    parent beforeCreate => parent created => parent beforeMount => child beforeCreate => child created => child beforeMount => parent mounted
    子组件销毁时：
    parent beforeUpdate => child beforeDestroy => child destroyed => parent updated
    父组件销毁时：
    parent beforeDestroy => child beforeDestroy => child destroyed => parent destroyed

父组件的挂载完成，更新完成，销毁完成都依赖于子组件的状态变化完成

组件激活时： created => mounted => activated
组件失活时： deactivated
销毁失活的组件： beforeDestroy => destroyed
销毁激活的组件： deactivated

  2. 插槽v-slot

    1. 概念
      子组件中提供给父组件使用的一个占位符；用标签表示，父组件可以在这个占位符中填充任何模板代码，比如HTML、组件等，填充的内容会替换掉子组件的插槽标签（替换占位符）
    2. 作用
      让用户可以拓展组件，去更好地复用组件和对其做定制化处理
    3. 分类
    • 定义
      • 默认插槽：<slot></slot>
      • 具名插槽：<slot name="customButton"></slot>
      • 作用域插槽（可以访问子组件的数据）：<slot :row="row"></slot>
