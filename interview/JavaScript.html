# 作用域

  1. 概念：
    作用域是指程序源代码中定义变量的区域。
    作用域规定了如何查找变量，也就是确定当前执行代码对变量的访问权限。
    JavaScript 采用词法作用域（lexical scoping），也称为静态作用域。
    函数的作用域在函数定义（声明）的时候就决定了（声明函数代码的位置）

  2. 作用域链

  3. 代码示例

# 执行上下文

  1. 概念：
    当执行一个函数（一段代码）的时候，会进行准备工作，这里的“准备工作”（如：变量提升 or 函数提升），就叫做创建"执行上下文execution context"（请注意是执行/调用，而非创建/定义）

  2. 分类： 根据可执行代码（executable code）全局代码、函数代码、eval代码分为三类
    全局上下文、函数上下文、eval上下文

  3. 如何管理很多个执行上下文?
    JavaScript引擎创建执行上下文栈（Execution context stack，ECS）来管理执行上下文

  4. 创建和销毁

1   var scope = "global scope";
2   function checkscope(){
3     var scope = "local scope";
4     function f(){
5        return scope;
6     }
7   return f();
8   }
9   checkscope();

// 以下是伪代码
const ECS = [ globaleContext ]; // 创建执行上下文栈, 默认栈底有一个全局上下文
ECS.push(<checkscope> functionContext); // 第9行 调用checkscope时，将函数上下文<checkscope>压进栈中
ECS.push(<f> functionContext); // 第6行 调用f时，将函数上下文<f>压进栈中
ECS.pop(); // f执行完毕，函数上下文<f>出栈
ECS.pop(); // checkscope执行完毕，函数上下文<checkscope>出栈

# this指向

  1. 如何确定this指向
    函数内部中的 this 指向谁，不是在函数定义时决定的，而是在函数执行的时候决定的
    联系上文：函数定义时决定作用域，函数执行时决定this

  2. 改变this指向的方式
    三种：call、apply、bind
    区别：

    • call立即执行函数，call(新的this指向, 调用call方法的函数的参数列表)
    • apply立即执行函数，apply(目标this指向, [param1, param2, ...])
    • bind返回一个新的函数，不执行；bind(this指向, 原函数的参数列表)
    • 伪代码
    fn.call(this, param1, param2, ...)
fn.apply(this, [param1, param2, ...])
var newFunc = fn.bind(this, param1, param2, ...);
newFunc();

  3. 箭头函数中的this
    箭头函数本身没有this，函数体内的this指向，就是定义该函数时所在的作用域指向的对象（继承于外层作用域中的this）

  4. 代码思考题

var obj = {
  hi: function() {

    }
  },
  sayHi: function() {

      console.log(this);
      return () => {
        console.log(this);
      }
    }
  },
  say: () => {
    console.log(this);
  }
}

hi();

# 闭包（closure）

  1. 概念

    1. 即使创建它的上下文已经销毁，它仍然存在（比如，内部函数从父函数中返回）
    2. 在代码中引用了自由变量
    3. 翻译：某个函数执行完毕（出执行栈），他的内部变量还被人引用着，导致无法被垃圾回收

  2. 代码示例

var scope = "global scope";

function checkscope() {
  var scope = "local scope";

  function f() {
      return scope;
  }
  return f;
}

var foo = checkscope();
foo();
  1. 经典面试题
var data = [];

for (var i = 0; i < 3; i++) {
  data[i] = function () {
    console.log(i);
  };
}
// 打印3次3

解决：1、闭包 2、let块级作用域

var data = [];

for (var i = 0; i < 3; i++) {
  (data[i] = function () {
    console.log(i);
  })(i);
}

var data = [];

for (let i = 0; i < 3; i++) {
  data[i] = function () {
    console.log(i);
  };
}

  1. 优缺点
    1）优点

    • 可以从内部函数访问外部函数的作用域中的变量，且访问到的变量长期驻扎在内存中，可供之后使用
    • 避免变量污染全局
    • 把变量存到独立的作用域，作为私有成员存在
      2）缺点
    • 对内存消耗有负面影响。因内部函数保存了对外部变量的引用，导致无法被垃圾回收，增大内存使用量，所以使用不当会导致内存泄漏
    • 对处理速度具有负面影响。闭包的层级决定了引用的外部变量在查找时经过的作用域链长度
    • 可能获取到意外的值(captured value)

# 原型、原型链

  1. 在规范里，prototype 被定义为：给其它对象提供共享属性的对象。 也就是说，prototype 自己也是对象，只是被用以承担某个职能罢了。 当某个对象，承担了为其它对象提供共享属性的职责时，它就成了其它对象的 原型（prototype）。

  2. 每个函数都有 prototype 属性，除了 Function.prototype.bind()，该属性指向原型。

function Foo() {}
Foo.prototype // { constructor: f Foo() }
Foo.prototype = { a: 1 } 
Foo.prototype // { a: 1 }
一旦我们改变了函数的 prototype，那么新对象就没有constructor这个属性了

constructor的作用

  • 让实例对象知道是什么函数构造了它
  • 如果想给某些类库中的构造函数增加一些自定义的方法，就可以通过 xx.constructor.method 来扩展
function Bar() {}
var bar = new Bar();
bar.constructor // ƒ Bar() {}
  1. 每个对象都有 __proto__ 属性，指向了创建该对象的构造函数的原型（对象）。其实这个属性指向了对象自身的 [[prototype]]属性，但是 [[prototype]] 是内部属性，我们并不能访问到，所以使用 __proto__ 来访问。
function Bar() {}
var bar = new Bar();
bar.__proto__ // { constructor: ƒ Bar() }
bar.__proto__ === Bar.prototype // true
bar.__proto__ === bar.constructor.prototype // true

对象可以通过 __proto__ 来寻找不属于该对象本身的属性，__proto__ 将对象连接起来组成了原型链。 （因为在 JSES6之前 中是没有类的概念的，为了实现类似继承的方式，通过 __proto__ 将对象和原型对象（给其他对象提供属性方法）联系起来组成原型链，得以让对象可以访问到不属于自己的属性。）

  1. 总结
  • Object 是所有对象的爸爸，所有对象都可以通过 __proto__ 找到它
  • Function 是所有函数的爸爸，所有函数都可以通过 __proto__ 找到它
  • Function.prototype 和 Object.prototype 是两个特殊的对象，他们由引擎来创建
  • 除了以上两个特殊对象，其他对象都是通过构造器 new 出来的
  • 函数的 prototype 是一个对象，也就是原型；因为原型本身也是一个对象，所以也有 __proto__
  • 对象的 __proto__ 指向原型， __proto__ 将对象和原型连接起来组成了原型链

# 继承

  1. 继承的方式
    1. 原型链继承
    2. 构造函数继承
    3. 组合继承
    4. 寄生组合继承

原型对象的用途是为每个实例对象存储共享的方法和属性，它仅仅是一个普通对象而已。 在构造函数中：为了属性(实例基本属性)的私有性、以及方法(实例引用属性)的复用、共享。我们提倡：

  • 将属性封装在构造函数中
  • 将方法定义在原型对象上
  1. 原型链继承（实现了子类对父类方法的继承）
Father.prototype.sayHi = function () {
  console.log('你好');
}

Child.prototype = new Father(); // 核心

const c1 = new Child();
c1.sayHi(); // 你好
  1. 构造函数继承（实现了子类对父类属性的继承）
function Parent(name) {
  this.name = name;
}

function Child(name, age) {
  // 继承父类的属性
  Parent.call(this, name); // 核心
  // 子类自己的属性
  this.age = age;
}

const c1 = new Child('张三', 18);
console.log(c1.name, c1.age); // 张三 18
  1. 组合继承（优化版）
function Parent(name) {
  this.name = name;
}
Parent.prototype.say = function () {
  console.log('你好');
}

function Child(name, age) {
  // A、继承父类属性
  Parent.call(this, name);
  this.age = age;
}

// 优化前的写法
// Child.prototype = new Parent(); // 导致了调用两次父类构造方法

// 优化后的写法
// B、继承父类方法，且避免了调用父类构造函数两次；缺点：子类constructor错误
Child.prototype = Parent.prototype;

const c1 = new Child('张三', 18);

console.log(c1.constructor); // Parent

// C、修复 constructor 丢失问题
Child.prototype.constructor = Child;

console.log(c1.constructor); // Child

// 存在问题：父类的原型对象被改掉，父类实例无法正确找到其构造函数
const p1 = new Parent('爸爸');
console.log(p1.constructor); // Child
  1. 寄生组合继承
function Parent(name) {
  this.name = name;
  this.hobbies = ['足球'];
}
Parent.prototype.say = function () {
  console.log('你好');
}

function Child(name, age) {
  // 1、继承父类属性
  Parent.call(this, name);
  this.age = age;
}
// 2、继承父类方法
// 通过创建中间对象,子类原型和父类原型,就会隔离开
Child.prototype = Object.create(Parent.prototype); // 核心

const c1 = new Child('张三', 18);
const p1 = new Parent('爸爸');

// 修复 constructor 丢失问题
Child.prototype.constructor = Child;

console.log(c1.constructor); // Child
console.log(p1.constructor); // Parent

// 拓展 XXX:
// Object.create()方法创建一个新对象,使用现有的对象来提供新创建的对象的__proto__
// 返回值： 一个新对象，带着指定的原型对象和属性
// 其中，o 是新创建对象的原型(对象)
// 原理：找中间人(新的对象)去获取Parent.prototype上的方法，且不影响Parent.prototype本身
function create(o) {
  function F() {}
  F.prototype = o;
  return new F();
}

# new关键字

new 操作符做的四件事
1、新生成了一个对象
2、链接到原型
3、绑定 this
4、返回新对象

function new() {
  // 创建一个空的对象
  let obj = new Object();
  // 获得构造函数(实参列表第一项) 
  let Con = [].shift.call(arguments);
  // 链接到原型 （使得 实例对象可以使用原型对象上的公共方法）
  obj.__proto__ = Con.prototype;
  // 绑定 this，执行构造函数 （调用构造函数方法 初始化各种实例属性）
  let result = Con.apply(obj, arguments);
  // 确保 new 出来的是个对象
  return typeof result === 'object' ? result : obj;
}

# 深拷贝、浅拷贝

  1. 区别
    深拷贝和浅拷贝，对基本数据类型的拷贝都是重新复制一份，但是对于引用数据类型，浅拷贝只会复制引用数据的地址，而深拷贝则需要在堆内存中开辟空间，重新复制一份一模一样的数据。
// 伪代码
var obj1 = {
  name: '张三',
  hobbies: ['踢球', '跑步'],
};

var obj2 = shallowClone(obj1); // 浅拷贝方法
obj2.hobbies === obj1.hobbies // true 新旧对象还是共享同一块内存

var obj3 = deepClone(obj1); // 深拷贝方法
obj3.hobbies === obj1.hobbies // false 新对象跟原对象不共享内存

  1. 浅拷贝的实现方式

  • ES6 扩展运算符 ...
  1. 深拷贝的实现方式

function deepClone(obj, hash = new WeakMap()) {
  if (obj === null) return obj; // 如果是null或者undefined我就不进行拷贝操作
  if (obj instanceof Date) return new Date(obj);
  if (obj instanceof RegExp) return new RegExp(obj);
  // 可能是对象或者普通的值  如果是函数的话是不需要深拷贝
  if (typeof obj !== "object") return obj;
  // 是对象的话就要进行深拷贝
  if (hash.get(obj)) return hash.get(obj);
  let cloneObj = new obj.constructor();
  // 找到的是所属类原型上的constructor,而原型上的 constructor指向的是当前类本身
  hash.set(obj, cloneObj);
  for (let key in obj) {
    if (obj.hasOwnProperty(key)) {
      // 实现一个递归拷贝
      cloneObj[key] = deepClone(obj[key], hash);
    }
  }
  return cloneObj;
}
let obj = { name: 1, address: { x: 100 } };
obj.o = obj; // 对象存在循环引用的情况

obj.address.x = 200;
console.log(d);

# 异步编程

  1. JavaScript异步编程的发展历程：
  • 第一个阶段 - 回调函数，但会导致两个问题:
    • 缺乏顺序性： 回调地狱导致的调试困难，和大脑的思维方式不符
    • 缺乏可信任性： 控制反转导致的一系列信任问题
  • 第二个阶段 - Promise，Promise是基于PromiseA+规范的实现，它很好的解决了控制反转导致的信任问题，将代码执行的主动权重新拿了回来。
  • 第三个阶段 - 生成器函数Generator，使用Generator，可以让我们用同步的方式来书写代码，解决了顺序性的问题，但是需要手动去控制next(...)，将回调成功返回的数据送回JavaScript主流程中。
  • 第四个阶段 - Async/Await，Async/Await结合了Promise和Generator，在await后面跟一个Promise，它会自动等待Promise的决议值，解决了Generator需要手动控制next(...)执行的问题，真正实现了用同步的方式书写异步代码。
